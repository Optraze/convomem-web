Customers are the core identity entity in ConvoMem. All memories and conversations are
scoped to a customer.

## Lookup

Find a customer by email, phone, external ID, or customer ID. This is the most common
way to identify a customer at the start of a conversation.

<CodeTabs>
  <CodeTab language="curl" filename="Terminal">
  ```bash
  curl -s -H "X-API-Key: $CONVOMEM_API_KEY" \
    "https://api.convomem.com/api/v1/customers/lookup?email=alice@example.com"
  ```
  </CodeTab>
  <CodeTab language="typescript" filename="lookup.ts">
  ```typescript
  const result = await client.customers.lookup({
    email: 'alice@example.com',
  })

  if (result.found) {
    console.log('Customer:', result.customer.id)
  }
  ```
  </CodeTab>
  <CodeTab language="python" filename="lookup.py">
  ```python
  from convomem import CustomerLookupParams

  result = client.customers.lookup(
      CustomerLookupParams(email="alice@example.com")
  )

  if result.found:
      print(f"Customer: {result.customer.id}")
  ```
  </CodeTab>
  <CodeTab language="rust" filename="main.rs">
  ```rust
  use convomem::CustomerLookupParams;

  let result = client.lookup_customer(
      &CustomerLookupParams {
          email: Some("alice@example.com".into()),
          ..Default::default()
      }
  ).await?;

  if result.found {
      println!("Customer: {}", result.customer.unwrap().id);
  }
  ```
  </CodeTab>
</CodeTabs>

## Create

Create a new customer profile. You can provide any combination of identity fields.

<CodeTabs>
  <CodeTab language="curl" filename="Terminal">
  ```bash
  curl -s -X POST -H "X-API-Key: $CONVOMEM_API_KEY" \
    -H "Content-Type: application/json" \
    "https://api.convomem.com/api/v1/customers" \
    -d '{"name": "Alice", "email": "alice@example.com"}'
  ```
  </CodeTab>
  <CodeTab language="typescript" filename="create.ts">
  ```typescript
  const customer = await client.customers.create({
    name: 'Alice',
    email: 'alice@example.com',
  })

  console.log('Created:', customer.id)
  ```
  </CodeTab>
  <CodeTab language="python" filename="create.py">
  ```python
  from convomem import CustomerCreateRequest

  customer = client.customers.create(
      CustomerCreateRequest(name="Alice", email="alice@example.com")
  )

  print(f"Created: {customer.id}")
  ```
  </CodeTab>
  <CodeTab language="rust" filename="main.rs">
  ```rust
  use convomem::CustomerCreateRequest;

  let customer = client.create_customer(
      &CustomerCreateRequest {
          name: Some("Alice".into()),
          email: Some("alice@example.com".into()),
          ..Default::default()
      }
  ).await?;

  println!("Created: {}", customer.id);
  ```
  </CodeTab>
</CodeTabs>

## Stats

Get aggregate memory and conversation statistics for your organization.

<CodeTabs>
  <CodeTab language="curl" filename="Terminal">
  ```bash
  curl -s -H "X-API-Key: $CONVOMEM_API_KEY" \
    "https://api.convomem.com/api/v1/customers/stats"
  ```
  </CodeTab>
  <CodeTab language="typescript" filename="stats.ts">
  ```typescript
  const stats = await client.getStats({ from: '2026-01-01' })

  console.log('Total customers:', stats.totalCustomers)
  console.log('Total memories:', stats.totalMemories)
  console.log('Active last 7d:', stats.active7d)
  console.log('Sentiment split:', stats.positive, stats.neutral, stats.negative)
  ```
  </CodeTab>
  <CodeTab language="python" filename="stats.py">
  ```python
  stats = client.customers.get_stats(from_date="2026-01-01")

  print(f"Total customers: {stats.total_customers}")
  print(f"Total memories: {stats.total_memories}")
  print(f"Active last 7d: {stats.active_7d}")
  ```
  </CodeTab>
  <CodeTab language="rust" filename="main.rs">
  ```rust
  let stats = client.get_customer_stats(Some("2026-01-01"), None).await?;

  println!("Total customers: {}", stats.total_customers);
  println!("Total memories: {}", stats.total_memories);
  ```
  </CodeTab>
</CodeTabs>

**Stats fields:**

| Field | Type | Description |
| --- | --- | --- |
| `totalCustomers` | number | All customer profiles |
| `totalMemories` | number | All memories across all customers |
| `avgMemories` | number | Average memories per customer |
| `totalConversations` | number | All conversation sessions |
| `active7d` | number | Customers with activity in the last 7 days |
| `positive` | number | Customers with positive sentiment |
| `neutral` | number | Customers with neutral sentiment |
| `negative` | number | Customers with negative sentiment |

## Merge candidates

ConvoMem detects customer profiles that may be duplicates and flags them for review.

<CodeTabs>
  <CodeTab language="curl" filename="Terminal">
  ```bash
  # List candidates
  curl -s -H "X-API-Key: $CONVOMEM_API_KEY" \
    "https://api.convomem.com/api/v1/customers/merge-candidates"

  # Dismiss a false positive
  curl -s -X POST -H "X-API-Key: $CONVOMEM_API_KEY" \
    "https://api.convomem.com/api/v1/customers/cust_123/merge-candidates/cust_456/dismiss"
  ```
  </CodeTab>
  <CodeTab language="typescript" filename="merge.ts">
  ```typescript
  const candidates = await client.listMergeCandidates()

  for (const { customer, candidate, similarity } of candidates) {
    console.log(`${customer.email} ↔ ${candidate.email} (${similarity?.toFixed(2)})`)
  }

  // Dismiss a false positive
  await client.dismissMergeCandidate('cust_123', 'cust_456')
  ```
  </CodeTab>
  <CodeTab language="python" filename="merge.py">
  ```python
  candidates = client.customers.list_merge_candidates()

  for c in candidates:
      print(f"{c.customer.email} ↔ {c.candidate.email} ({c.similarity:.2f})")

  # Dismiss
  client.customers.dismiss_merge_candidate("cust_123", "cust_456")
  ```
  </CodeTab>
  <CodeTab language="rust" filename="main.rs">
  ```rust
  let candidates = client.list_merge_candidates().await?;

  for c in &candidates {
      println!("{:?} ↔ {:?}", c.customer.email, c.candidate.email);
  }

  // Dismiss
  client.dismiss_merge_candidate("cust_123", "cust_456").await?;
  ```
  </CodeTab>
</CodeTabs>

## Handoff

Generate a full cross-channel briefing — conversation journey, key memories, sentiment
trend, open-issue detection, and AI-written narrative — for a human agent taking over.

See the dedicated [Handoff](/docs/handoff) page for the full reference.

## Response fields

| Field | Type | Description |
| --- | --- | --- |
| `id` | string | Unique customer identifier |
| `name` | string | Customer name |
| `email` | string | Email address |
| `phone` | string | Phone number |
| `externalId` | string | External system identifier |
| `metadata` | object | Custom metadata |
| `createdAt` | string | ISO-8601 creation timestamp |
| `updatedAt` | string | ISO-8601 last update timestamp |
