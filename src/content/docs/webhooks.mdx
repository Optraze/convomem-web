Webhooks let your systems react the moment ConvoMem fires an event ‚Äî without polling.
Webhook endpoints are configured in the **dashboard** (Settings ‚Üí Webhooks). They cannot
be managed via an org API key.

## Live events

| Event | Fires when | Key `data` fields |
| :-- | :-- | :-- |
| `memory.captured` | Background extraction commits memories for a capture | `customerId`, `saved`, `skipped`, `captureId`, `conversationId` |
| `conversation.completed` | A conversation is ended | `conversationId`, `customerId`, `outcome` |
| `conversation.escalated` | A conversation is escalated | `conversationId`, `customerId`, `reason` |

> `customer.created` appears in the event catalog but is **not currently dispatched** at
> runtime ‚Äî don't build on it yet.

## Event payload

Every delivery is a `POST` with an `X-ConvoMem-Signature` header:

```json title="memory.captured"
{
  "event": "memory.captured",
  "createdAt": "2026-06-17T10:00:00.000Z",
  "data": {
    "customerId": "f3b39d1b-2522-4a00-98df-8cdde783ba12",
    "conversationId": "87c53d10-85f8-4447-920f-b1e847c23a1a",
    "captureId": "cap_xyz789",
    "saved": 3,
    "skipped": 0
  }
}
```

## Verifying signatures

The signature is `sha256=<HMAC-SHA256(rawBody, webhookSecret)>`. Always verify before
processing ‚Äî anyone who learns your URL could otherwise POST to it.

Compute the HMAC over the **raw bytes** of the request body, not a re-serialized object.

```typescript title="verify.ts"
import crypto from 'node:crypto'

function verifyWebhook(
  rawBody: Buffer | string,
  signatureHeader: string,
  secret: string,
): boolean {
  const sig = signatureHeader.replace('sha256=', '')
  const digest = crypto
    .createHmac('sha256', secret)
    .update(rawBody)
    .digest('hex')
  const a = Buffer.from(sig, 'hex')
  const b = Buffer.from(digest, 'hex')
  return a.length === b.length && crypto.timingSafeEqual(a, b)
}
```

```python title="verify.py"
import hmac, hashlib

def verify_webhook(raw_body: bytes, signature_header: str, secret: str) -> bool:
    expected = signature_header.replace("sha256=", "")
    computed = hmac.new(secret.encode(), raw_body, hashlib.sha256).hexdigest()
    return hmac.compare_digest(expected, computed)
```

<Callout type="warning" title="Always verify">
Unverified webhook payloads should be rejected ‚Äî anyone who learns your URL could
otherwise POST to it.
</Callout>

## Delivery semantics

- **Retries** ‚Äî non-`2xx` or timeout triggers up to **3 retries** with exponential backoff.
- **Idempotency** ‚Äî deliveries can repeat; de-duplicate on `(event, conversationId)` or `captureId`.
- **SSRF protection** ‚Äî destination URLs must be `https://`. Loopback and cloud metadata addresses are rejected.
- **Timeout** ‚Äî your endpoint must respond within 10 s.
