Retrieval is where memory pays off. At the start of a new interaction, ask ConvoMem for
the context most relevant to what's happening now.

## Basic retrieval

```typescript title="retrieve.ts"
import { ConvoMem } from '@optraze/convomem-sdk'

const client = new ConvoMem({ apiKey: process.env.CONVOMEM_API_KEY! })

const { context, memories, tokenCount } = await client.lookup(
  'preferred shipping method',
  { email: 'cust@example.com' },
)
```

The `topic` drives semantic ranking — ConvoMem returns the memories most relevant to that
topic. You get back:

- `context` — a newline-separated list of memories, pre-formatted and ready to drop into a
  system prompt
- `memories` — the raw `Memory[]` array if you need structured access
- `tokenCount` — approximate size of the `context` string

## Injecting into a prompt

Most teams drop the `context` string directly into a system prompt:

```typescript title="prompt.ts"
const { context } = await client.lookup(
  userMessage,
  { customerId: 'cust_uuid_123' },
)

const system = `You are a support agent. Known context about this customer:\n${context}`
```

<Callout type="tip" title="Keep it tight">
The lookup is capped at 10 memories / 800 tokens. Ranking weights:
`relevance × 0.5 + importance × 0.3 + recency × 0.2`. You get high-signal context,
not an exhaustive dump.
</Callout>

## Listing all memories

To browse all stored memories for a customer with pagination, use `listMemories`:

```typescript title="list.ts"
const { memories, total } = await client.listMemories(
  { customerId: 'cust_uuid_123' },
  { page: 1, limit: 20, category: 'preference' },
)

for (const mem of memories) {
  console.log(`- ${mem.content} (${mem.category})`)
}
```
